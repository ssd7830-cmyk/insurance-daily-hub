# 보험설계사 데일리뉴스 대시보드 기획서

> 작성일: 2026-06-16
> 프로젝트 폴더: `고창환/`

---

## 1. 프로젝트 개요

보험설계사가 매일 아침 출근해서 **단 한 화면**으로 그날의 보험 업계 동향을 파악할 수 있게 하는 웹 대시보드.
관련 뉴스를 자동 수집·분류해 보여주고, 고객 상담·영업에 바로 활용할 수 있는 인사이트를 제공한다.

### 핵심 가치
- **시간 절약**: 여러 사이트를 돌아다닐 필요 없이 한 화면에서 끝.
- **맞춤형**: 생명보험·손해보험·세금·정책 등 설계사가 실제 쓰는 주제 중심.
- **영업 활용**: "오늘의 상담 포인트" 형태로 고객에게 바로 전달 가능.

|------|------|

## 3. 주요 기능

### 3.1 데일리 뉴스 피드 (핵심)
- 보험 관련 뉴스 자동 수집 (네이버 뉴스 API 또는 언론사 RSS)
- 발행 시간순 / 인기순 정렬
- 제목, 요약, 출처, 발행시각, 원문 링크 표시

### 3.2 카테고리 분류
- 생명보험 / 손해보험 / 건강·실손 / 세금·연금 / 정책·법규 / 회사동향
- 키워드 기반 자동 태깅

### 3.3 키워드 트렌드
- 오늘 가장 많이 등장한 키워드 Top 10
- 워드클라우드 또는 막대 차트 시각화

### 3.4 오늘의 상담 포인트
- 주요 기사에서 "고객에게 이렇게 말하세요" 형태의 한 줄 멘트 추출
- 복사 버튼으로 카톡·문자에 바로 활용

### 3.5 즐겨찾기 / 스크랩
- 관심 기사 저장
- 나중에 다시 보기

## 5. 기술 스택

| 영역 | 기술 | 비고 |
|------|------|------|
| 백엔드 | Python + Flask | 가볍고 빠른 구현 |
| 뉴스 수집 | requests / feedparser | 네이버 API 또는 RSS |
| 데이터 저장 | SQLite 또는 JSON | `data/` 폴더 |
| 프론트엔드 | HTML + CSS + JS | `templates/`, `static/` |
| 차트 | Chart.js | 키워드 트렌드 시각화 |
| 스케줄링 | APScheduler | 주기적 자동 수집 |

### 폴더 구조 (예정)

## 6. 데이터 수집 방식

### 옵션 A — 네이버 검색 API (권장)
- '보험' 관련 키워드로 뉴스 검색
- 필요: 네이버 개발자센터 Client ID / Secret 발급
- 장점: 안정적, 정렬·페이징 지원

### 옵션 B — 언론사 RSS
- 보험 전문지(보험신문 등) RSS 구독
- 장점: API 키 불필요
- 단점: 보험 전용 필터링 직접 구현 필요

> 초기에는 **샘플 더미 데이터**로 화면을 완성한 뒤, 실제 소스를 연결하는 방식 권장.

## 7. 개발 단계 (로드맵)

> 실제 스택: **Vite + React + Tailwind / 서버리스·무DB / 날짜별 정적 JSON / Claude Haiku 큐레이션**

| 단계 | 작업 | 상태 |
|------|------|:---:|
| 1단계 | 기획 확정 (본 문서) | ✅ |
| 2단계 | RSS 피드 실측 검증 + `sources.json` (국내 4·해외 3) | ✅ |
| 3단계 | `collect.js` 수집→필터→dedup→5축 큐레이션→날짜별 JSON | ✅ |
| 4단계 | 대시보드 UI (사이드바·국내/해외·날짜변경·점수배지) | ✅ |
| 5단계 | 큐레이션 기준 공개 패널 (5축 가중치·컷 규칙·통계) | ✅ |
| 6단계 | Vercel 배포로 고정 URL 확보 | ⏳ |
| 7단계 | 설계사 부가 기능 확장 (아래 §9) | 📋 |

## 8. 큐레이션 기준 (뉴스 질 관리)

홍보성 기사를 걸러내고 "설계사에게 진짜 필요한 뉴스"만 남기기 위한 5축 채점.

| 축 | 가중치 | 핵심 질문 |
|----|:---:|------|
| 업무 관련성 | 30% | 내 영업·상품·고객과 직결되는가 |
| 영업 활용도 | 25% | 고객 상담에 바로 쓸 멘트가 나오는가 |
| 시의성 | 20% | 지금 알아야 할 제도·정책 변화인가 |
| 영향 범위 | 15% | 다수 고객·업계 전체에 영향을 주는가 |
| 신뢰·밀도 | 10% | 사실 기반이고 정보가 충실한가 |

- **컷:** 45점 미만 제외 + 홍보성(MOU·봉사활동·인사·세미나·광고·부고) 제외
- **저작권:** 기사 전문 복제 금지, 요약은 AI가 100% 재서술(해외는 번역 포함), 원문은 링크로만

## 9. 설계사용 기능 전체 지도

> 뉴스는 "매일 여는 이유"일 뿐. 출근→상담→마감까지 하루 동선을 받치는 기능 후보.

### 9-1. 🏠 데일리 허브 *(매일 여는 이유)*
| 기능 | 설명 | 상태 |
|------|------|:---:|
| 오늘의 뉴스 큐레이션 | 국내/해외·카테고리·점수 | ✅ |
| 오늘의 상담 멘트 + 복사 | 뉴스에서 뽑은 고객용 한 줄 | 📋 |
| 오늘 할 일 / 알림 | 만기·갱신·생일 고객 자동 표시 | 📋 |
| 오늘의 일정 | 미팅·전화 약속 캘린더 | 📋 |
| 마감 체크리스트 | 청약·서류 임박 건 | 📋 |

### 9-2. 👥 고객 관리 (가벼운 CRM) — *로그인·DB 필요*
| 기능 | 설명 |
|------|------|
| 고객 카드 | 이름·연락처·가입상품·메모 |
| 보장 분석 / 리모델링 | 가입 현황 → 보장 공백 시각화 |
| 재접촉 알림 | 미연락·생일·만기 자동 푸시 |
| 상담 이력 | 통화·미팅 타임라인 |
| 가망고객 파이프라인 | 잠재→상담→청약 단계 보드 |

### 9-3. 🧮 상담 도구 (현장 즉시)
| 기능 | 설명 |
|------|------|
| 보험료 계산기 | 나이·성별·보장 → 예상 보험료 |
| 연금·노후 시뮬레이터 | 국민연금+사적연금 → 노후 월소득 (클로징 도구) |
| 절세 계산기 | 연금저축·IRP 세액공제, 비과세 한도 |
| 상품 비교표 | 회사별 같은 보장 한눈에 |
| 실손 세대 전환 비교 | 1~4세대 vs 5세대 유불리 (뉴스 연계) |

### 9-4. 📤 고객용 콘텐츠 (마케팅)
| 기능 | 설명 |
|------|------|

## 10. 우선순위 (먼저 붙일 것)

서버리스·무DB 제약에서 **즉시 효용 + 매일 사용** 기준 Top 3.

| 순위 | 기능 | 이유 |
|:---:|------|------|
| **1** | 오늘의 상담 멘트 + 복사 | 데이터(`why_matters`) 이미 있음 → 거의 공짜 |
| **2** | 뉴스 스크랩 / 즐겨찾기 | "고객에게 보낼 기사" 모으기 → 체류 이유 |
| **3** | 연금·노후 시뮬레이터 | 단독으로도 강력한 클로징 도구, 연금 뉴스와 시너지 |

> ⚠️ CRM(고객 카드)은 가치가 가장 크지만 **개인정보·로그인·DB**가 필요해 현재 "서버리스·무DB" 제약과 충돌 → 별도 단계(로그인 도입)로 분리.
